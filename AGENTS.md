

# 开发模式,有两种
- 默认模式:你直接自行规划,分步完成并自我校验,先分清楚是新增需求还是需求变更还是bug修改,并使用中文把需求文档整理到 /docs 目录下,以当前年月+处理类型+中文命名
- 工作流模式:包括新需求开发和需求调整,我会明确以`需求开发`或`需求调整`开头,你需要读取 agent-workflow.md 中的内容并严格遵循里面的规则进行开发
